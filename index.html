Git Version Control

Git [Global Information Tracker] is a free, open-source version control system that helps programmers track changes to their code files and projects


Here is the list of useful commands to follow:

  • U: U stands for untracked in Source code
  • mv: used to rename and move the files
  • touch: by using this command we can create multiple files
  • git init: Initializes a new Git repository in your project.
  • git log: Shows the history of commits.
  • git checkout master: used to reach back to the current version
  • git push: Pushes your commits to a remote repository (like GitHub).
  • git pull: Pulls changes from a remote repository.
  • git branch: Lists or creates branches in your project.
  • git merge <branch>: Merges a branch into the current branch.
  • git remote show origin: command used to check the connection of GitHub repository
  • git branch -M: command used to rename the branch name
  • git push -u origin "name": used to push a local branch(es) to a remote repository (origin).
  • git fetch: git fetch is a primary command used to download contents from a remote repository.
  • git merge origin/'branch name': merging the remote branch
